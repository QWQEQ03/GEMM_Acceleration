# 面向网络大模型注意力机制的通用矩阵乘法（GEMM）并行加速研究
### 2025140761 陈益睿

#### 一、 研究背景与选题意义

在网络大模型的科研与应用中，模型通常需要处理包含长周期的网络流量特征或复杂拓扑节点的长序列数据。在其核心的 Transformer 结构中，自注意力机制的核心计算密集型算子为通用矩阵乘法（GEMM），例如计算注意力权重矩阵及其与价值矩阵的乘积。

随着网络节点规模和上下文长度的增加，GEMM 的计算开销呈二次方增长，成为制约大模型推理与实时响应的关键瓶颈。本项目旨在利用高性能计算技术，基于 CUDA 针对特定维度的网络特征矩阵乘法进行并行化加速优化，探索底层算力释放的有效路径。这不仅能加深对 GPU 硬件架构及并行算法的理解，也能为后续网络大模型的高效部署提供底层加速经验。

#### 二、 主要研究内容与优化策略

本项目将从最基础的 CPU 串行矩阵乘法出发，逐步过渡到高效的 GPU 并行实现，主要包含以下层层递进的优化阶段：

1. **CPU 串行基线实现：** 使用标准 C++ 实现朴素的三重循环矩阵乘法，作为评估加速比的基准。
    
2. **CUDA 朴素并行加速：** 将独立的数据点计算映射到 GPU 的线程网格和线程块中，利用全局内存进行初级计算并行化。
    
3. **基于共享内存的分块优化：** 针对访存瓶颈，利用 GPU 的共享内存实现分块矩阵乘法。通过将子矩阵加载到高速的共享内存中，大幅减少对全局内存的重复访问，提升显存带宽利用率。
    
4. **访存合并与参数调优：** 优化线程块大小（如 $16 \times 16$ 或 $32 \times 32$），确保同一线程束对全局内存的访问满足合并访存条件，进一步降低访存延迟。
    

#### 三、 实验设计与性能评估

- **开发环境：** Linux (Ubuntu) 操作系统，C++ / CUDA 编程语言。
    
- **输入数据构建：** 模拟 NetLLM 在典型网络任务下的隐藏层维度（如构造大小为 $1024 \times 1024$、$2048 \times 2048$ 及 $4096 \times 4096$ 的单精度浮点数矩阵）。
    
- **评估指标：** * **执行时间（ms）：** 记录各版本算法的实际计算耗时。
    
    - **计算吞吐量（GFLOPS）：** 每秒十亿次浮点运算数，衡量硬件计算能力的发挥程度。
        
    - **加速比（Speedup）：**
        
        $$Speedup = \frac{T_{cpu}}{T_{gpu}}$$
        

#### 四、 预期成果

1. 交付一套结构清晰、带完整注释的 C++/CUDA 矩阵乘法加速代码。
    
2. 产出一份包含性能对比折线图/柱状图的最终大作业报告，预期在 GPU 分块优化下，相比单线程 CPU 基线取得数倍的加速比。
